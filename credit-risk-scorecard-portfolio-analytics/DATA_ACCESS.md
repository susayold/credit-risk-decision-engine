# Data Access

## Core Data

This repository is designed to run locally against the Project 0 data core.

Author local path example:

```text
<PROJECT_0_DATA_CORE>/data/gold/project3/project3_credit_risk_account_base.csv.gz
```

Portable repo path:

```text
./data/processed/formula_engine_account_sample.csv.gz
./data/processed/enriched_scorecard_challenger_sample.csv.gz
```

The full raw data is intentionally not committed to GitHub.

Important for recruiters:

- GitHub contains only a generated 100k account-level sample plus aggregated reports.
- Full raw public datasets are not stored in this repo because they are large.
- A recruiter can review the project logic from the included sample, reports, Excel workbook, visuals and HTML case study.
- A full rebuild requires recreating the Project 0 data core from the public sources below.

## Included Sample

The repository includes processed samples for inspection:

```text
data/processed/formula_engine_account_sample.csv.gz
data/processed/enriched_scorecard_challenger_sample.csv.gz
```

Sample size: up to 100,000 rows per sample.

These samples are generated by:

```bash
python scripts/01_build_formula_first_engine.py
python scripts/08_build_enriched_scorecard_challenger.py
```

## Light Rebuild Mode / Reviewer Quick-Run

A reviewer does **not** need to recreate Project 0 to inspect the portfolio logic.

Quick-run from the included sample:

```bash
python scripts/06_quick_review_from_sample.py
python scripts/02_validate_project3_outputs.py
```

What this proves:

- `scripts/06_quick_review_from_sample.py` reads `data/processed/formula_engine_account_sample.csv.gz`.
- It checks sample rows, matured rows, default rate, average formula PD, EAD proxy, expected loss and risk-grade summaries.
- It creates `reports/34_sample_quick_review.md`, `reports/34_sample_quick_review_summary.json`, `reports/34_sample_quick_review_by_grade.csv` and `reports/34_sample_quick_review_by_decile.csv`.
- `scripts/02_validate_project3_outputs.py` is portable and validates the included reports, docs, visuals, Excel workbook, repo index, website copy and enriched challenger benchmark.

Important boundary:

- The 100k sample is for inspection and quick sanity checks.
- It does **not** regenerate the full WOE/IV development, enriched challenger development, train/validation/test windows, stress testing, Power BI outputs or Project 0 data core.
- Full regeneration still requires the Project 0 data core described below.

| Source | Public link | Purpose |
|---|---|---|
| Lending Club granting model dataset | https://zenodo.org/records/11295916 | Core accepted/booked consumer lending accounts used for formula-first target, WOE/IV, PD, score bands and validation. |

| FRED UNRATE | https://fred.stlouisfed.org/series/UNRATE | Unemployment macro series for stress scenario context. |
| FRED CPIAUCSL | https://fred.stlouisfed.org/series/CPIAUCSL | CPI macro series for inflation context. |
| FRED FEDFUNDS | https://fred.stlouisfed.org/series/FEDFUNDS | Federal funds rate series for macro/rate context. |
| FRED DGS10 | https://fred.stlouisfed.org/series/DGS10 | 10-year Treasury yield series for macro/rate context. |

## Rebuild Reality

This Project 3 repo is not a one-click raw-data rebuild from GitHub alone. It is a portfolio deliverable built on top of Project 0.

To fully rebuild:

1. Download the public datasets above.
2. Recreate the Project 0 data core folder.

Author local path example:

```text
<PROJECT_0_DATA_CORE>
```

Or set a portable environment variable pointing to your own Project 0 folder:

```bash
set FINANCIAL_RISK_DATA_CORE=<path_to_project0_data_core>
```

PowerShell:

```powershell
$env:FINANCIAL_RISK_DATA_CORE = "<path_to_project0_data_core>"
```

3. Generate Project 3 gold tables inside.

Author local path example:

```text
<PROJECT_0_DATA_CORE>/data/gold/project3
```

4. Run Project 3 scripts:

```bash
python scripts/01_build_formula_first_engine.py
python scripts/03_build_completion_pack.py
python scripts/04_build_financial_modeling_summary.py
python scripts/06_quick_review_from_sample.py
python scripts/08_build_enriched_scorecard_challenger.py
python scripts/02_validate_project3_outputs.py
```

## Why Raw Data Is Excluded

Raw files are large and belong in the local data foundation, not in the GitHub repo. The repo publishes:

- Code
- Documentation
- Aggregated reports
- Visuals
- A processed 100k-row sample
- Reproducible instructions
